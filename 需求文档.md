# Minecraft 1.7.10 性能向自用 Client — 需求冻结 PRD & Backlog（v1.2）

> 本文档为“可执行、可验收、可变更”的需求冻结版本。  
> 目标：在固定验收环境下达到 **常态 ≥3000 FPS**、**仰望星空 ≥7000 FPS**，并实现 Freelook / Fog 编辑器 / HUD 不渲染 / 配置持久化 / OptiFine 双路径。

---

## 0. 合规边界（Non-Goals）
- 不实现任何优势/作弊向功能：例如 aim/auto-click/reach/ESP/velocity/绕过反作弊等
- 不做 replay
- 不考虑 Forge 兼容
- 不提供启动器、不改认证/登录流程（仅由第三方启动器启动）
- 项目交付仅用于本地自用（不二次分发）

---

## 1. 范围与平台

### 1.1 支持范围
- **仅 Minecraft 1.7.10**
- **仅 Windows**
- JDK：**8**
- 启动：**依赖第三方启动器**
- 交付形态：**A（自用本地环境）**

### 1.2 OptiFine 策略
- **自动检测并切换路径（策略 #3：双模式）**
  - 检测到 OptiFine：走 **兼容路径**
  - 未检测到 OptiFine：走 **极限路径**
- 允许为 FPS 强制选择更快渲染路径，并在两条路径间严格隔离（避免互相污染）

---

## 2. 验收环境与固定设置（Benchmark 约束）

### 2.1 固定验收机
- 分辨率：1080p
- GPU：RTX 4060
- CPU：i5-12600KF
- 物理内存：32GB
- JVM 堆：**2–4GB**（为 FPS 优化，允许调整 GC 参数）
- 显示器刷新率：**180Hz**（记录项，不锁帧）

### 2.2 固定验收设置（必须强制）
- 显示模式：**Exclusive Fullscreen**
- VSync：**强制关闭**
- Max Framerate：**强制 Unlimited**
- Render Distance：**2 chunks**
- 强制关闭（硬开关，直接不走渲染路径 / 早返回）：
  - Clouds（云）
  - Dynamic Lighting（动态光照）
  - Smooth Lighting（平滑光照）
  - Animations（动画，全关）
  - Particles（粒子：**全部不渲染，无白名单**）
  - Entity Shadows（实体阴影）
- Fog（雾）：保留默认逻辑为基线，但允许编辑器覆盖（见 4.2）
- 贴图/过滤：
  - 强制关闭 Mipmap / Anisotropic Filtering（为极限 FPS）

### 2.3 固定基准场景
- 单人创造 / 超平坦
- 固定坐标、固定时间/天气（建议固定：白天、无雨）
- 固定相机角度与位置（避免波动）

---

## 3. 性能验收标准（Definition of Done）

在 **2. 的所有约束**成立时：

1) 常态场景：**平均 FPS ≥ 3000**
2) “仰望星空”场景：**平均 FPS ≥ 7000**
   - 定义：相机仰角接近 90°，最小化世界几何/实体渲染负载（由 Benchmark Harness 固定）

稳定性验收（避免“高 FPS 但抖动/泄漏”）：
- 连续运行 **10 分钟**：
  - 不出现明显帧时间尖峰（建议记录 P95/P99/P99.9 frame time）
  - 堆内存无持续性上升（无明显泄漏/缓存失控）
  - GC 暂停不可出现“肉眼可感”的频繁卡顿

---

## 4. 功能需求（只做这些）

### 4.1 Freelook（Hold）
- 触发：**按住启用（Hold）**
- 输入优先级：Freelook > 其他（当键位冲突时）
- GUI 打开时：**不启用**（避免输入冲突）
- 行为：启用时相机可自由旋转，但玩家真实朝向/移动逻辑不被相机旋转影响
- 松开：相机立即回到玩家真实朝向（当前 v1 固定；若需平滑回正可走变更单）

### 4.2 Fog 编辑器（全维度 + 覆盖允许 + 热更新）
- 基线：Minecraft 1.7.10 默认雾逻辑（维度/天气/水下/失明等）
- 生效范围：**所有维度**
- 覆盖：允许覆盖水下雾、失明等特殊雾
- 参数：至少提供 Fog Start / Fog End
- 更新方式：**热更新（拖动滑条立即生效）**
- 允许：极近雾（“最近仅可见脚下方块”效果）
- 第三人称：允许极近雾生效

### 4.3 HUD 不渲染（只跳过绘制）
- Chat（聊天栏）：**不绘制**（数据仍接收/解析）
- Scoreboard（计分板）：**不绘制**（数据仍接收/解析）

### 4.4 配置持久化（JSON）
- 格式：JSON（可读、可 diff、易扩展）
- 保存时机：**退出游戏 / 关闭设置界面时写盘**
- 必备：配置版本号（便于未来扩展）

### 4.5 恢复默认（最高优先级）
- 键位：**K**
- 优先级：**最高**（任何状态都应可触发）
- 行为：立即恢复所有可配置项到默认值，并立即生效（尤其 Fog / HUD 隐藏）

### 4.6 字体渲染（允许重写/替换）
- 允许为了性能进行字体渲染路径瘦身/替换
- 目标：设置 UI 与少量文本展示的渲染成本极低

### 4.7 UI 风格
- 主题风格：Anthropic 官网风格（极简、留白、克制动效）
- UI 范围：仅覆盖本项目必要设置面板（Freelook 键位、Fog、HUD 隐藏、恢复默认提示、性能状态）

---

## 5. 关键设计决策（已确认）
- 多线程范围：**B（渲染提交/批处理优化，主线程驱动，减少状态切换）**
- 不做：世界/实体 tick 并行（高风险并发）
- OptiFine：自动检测 + 双路径隔离
- 贴图过滤：强制关闭 mipmap/AF

---

## 6. 工程 Backlog（可执行任务拆分）

### M0：Benchmark Harness（必须先做，否则优化不可控）
**目标：可复现、可回归、可对比。**

- M0.1 固定世界与环境
  - 自动创建/加载超平坦世界
  - 固定坐标、时间、天气
- M0.2 固定两种相机场景
  - 常态视角（固定 yaw/pitch）
  - 仰望星空（固定 pitch≈-90°）
- M0.3 指标采样与输出
  - 平均 FPS（固定采样窗口，如 30s/60s）
  - frame time 分位数：P95/P99/P99.9
  - GC 次数/暂停时间（如能获取）
  - 堆占用趋势（定时采样）
  - 输出：本地日志（建议 JSON + 人类可读摘要）

**M0 验收：**
- 同一构建重复跑 3 次，平均 FPS 波动 < 5%（建议目标），否则继续固定条件直到稳定。

---

### M1：极限性能裁剪（冲 3000/7000 的核心）
- M1.1 粒子系统裁剪
  - 目标：粒子 **不渲染**；优先跳过绘制，再尽量减少上游生成/更新成本
- M1.2 强制禁用项硬开关
  - 云/动画/实体阴影/光照相关路径：早返回、减少 GL 状态切换与 draw calls
- M1.3 HUD：chat/scoreboard 不绘制
  - 只短路 draw，不影响数据处理
- M1.4 Fog 参数注入 + 热更新
  - 全维度 + 覆盖特殊雾来源
- M1.5 字体渲染瘦身/替换
  - 以设置 UI 为核心，减少阴影/复杂布局成本（实现细节由工程决定）

**M1 验收：**
- 在验收环境下达成或逼近 3000/7000 目标
- 连续 10 分钟稳定性达标（抖动/泄漏/GC）

---

### M2：OptiFine 双路径 + UX 收敛
- M2.1 OptiFine 自动检测
- M2.2 双路径隔离架构
  - OF 兼容路径：尽量不破坏 OF 替换链路
  - 极限路径：最大化裁剪与跳过
- M2.3 设置 UI（Anthropic 风格）
  - Freelook 键位
  - Fog Start/End（滑条热更新）
  - HUD 隐藏（chat/scoreboard）
  - 恢复默认（提示 K）
- M2.4 JSON 持久化（关设置/退出写盘）
- M2.5 K 键恢复默认（最高优先级 & 任何状态可用）

**M2 验收：**
- 有 OptiFine / 无 OptiFine 两种运行方式均可用
- 两种模式均满足性能与稳定性 DoD
- K 键在任何状态可救场

## 7. Benchmark 流程建议（供实现 Harness 使用）
> 具体实现可因工程而异，但必须保证“可复现”。

- 每次跑分顺序：
  1) 进入固定世界 → 传送到固定坐标 → 固定时间/天气
  2) 等待 5s 预热（JIT/纹理等）
  3) 常态视角采样 60s → 输出统计
  4) 仰望星空视角采样 60s → 输出统计
  5) 连续稳定性跑 10min（任选常态或两者交替）→ 输出趋势

- 记录项：
  - 构建版本号 / Git commit（如有）
  - 是否检测到 OptiFine（true/false）
  - JVM 参数（Xmx/Xms/GC）
  - 运行模式（极限/兼容）
